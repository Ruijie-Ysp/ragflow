# 向量数据展示功能使用指南

## 功能概述

RAGFlow 现在支持在前端查看文档 chunk 的向量数据和元数据，帮助你更好地理解和分析向量化的文本内容。

## 如何使用

### 方式 1：通过前端界面查看（推荐）

1. **进入知识库**
   - 登录 RAGFlow
   - 进入你的知识库
   - 选择一个已经解析完成的文档

2. **打开 Chunk 管理页面**
   - 点击文档，进入 Chunk 列表页面
   - 在工具栏中找到 **函数图标按钮** (fx)

3. **查看向量数据**
   - 点击函数图标按钮，打开"向量数据查看器"模态框
   - 模态框包含三个标签页：
     - **向量统计**：显示所有 chunk 的向量数据汇总统计
     - **Chunk 列表**：显示每个 chunk 的向量元数据
     - **详细信息**：点击某个 chunk 后，显示该 chunk 的完整向量信息

4. **查看向量元数据**
   
   每个 chunk 的向量元数据包括：
   - **向量维度**：通常为 1024 维
   - **向量范数**：向量的 L2 范数（欧几里得范数）
   - **均值**：向量所有元素的平均值
   - **标准差**：向量元素的标准差
   - **最小值/最大值**：向量中的最小和最大值
   - **字段名称**：向量在数据库中的字段名（如 `q_1024_vec`）

5. **下载向量数据**
   
   - 在模态框顶部，启用"包含完整向量"开关
   - 点击"下载所有向量"按钮
   - 系统会下载一个 JSON 文件，包含所有 chunk 的完整向量数据

### 方式 2：通过 API 调用

#### 内部 API（需要登录 Token）

```bash
curl -X POST http://localhost/api/chunk/list \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_LOGIN_TOKEN" \
  -d '{
    "doc_id": "你的文档ID",
    "page": 1,
    "size": 10,
    "include_vector": true,
    "include_full_vector": false
  }'
```

#### 外部 API（使用 API Token）

```bash
curl -X POST http://localhost/api/list_chunks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -d '{
    "doc_id": "你的文档ID",
    "include_vector": true,
    "include_full_vector": false
  }'
```

**参数说明**：
- `include_vector`: 是否返回向量元数据（统计信息）
- `include_full_vector`: 是否返回完整的向量数组（数据量大，慎用）

## 向量统计汇总

在"向量统计"标签页中，你可以看到：

1. **总块数**：文档中的总 chunk 数量
2. **包含向量**：有向量数据的 chunk 数量
3. **向量维度**：向量的维度（所有 chunk 相同）
4. **向量覆盖率**：有向量数据的 chunk 占比
5. **平均范数**：所有向量的平均范数
6. **平均均值**：所有向量均值的平均值
7. **平均标准差**：所有向量标准差的平均值

## 应用场景

### 1. 质量检查
- 检查向量范数是否在合理范围内
- 发现异常的向量数据（范数过大或过小）
- 验证向量化是否成功

### 2. 数据分析
- 分析不同文档的向量分布特征
- 比较不同 chunk 的向量相似度
- 导出向量数据进行进一步分析

### 3. 调试和优化
- 理解 embedding 模型的输出特征
- 优化检索参数
- 分析检索结果的向量相似度

## 性能建议

1. **分页加载**：
   - 默认每次加载 50 个 chunk
   - 如果文档很大，建议分批查看

2. **完整向量**：
   - 仅在需要时启用"包含完整向量"
   - 完整向量数据量大（1024 维 × 4 字节 ≈ 4KB/chunk）
   - 下载后建议保存到本地，避免重复请求

3. **缓存**：
   - 向量元数据会被前端缓存
   - 关闭模态框后再次打开会重新加载

## 技术细节

- **向量存储**：向量存储在 OpenSearch/Elasticsearch 中
- **字段格式**：`q_{dimension}_vec`，如 `q_1024_vec`
- **向量范数**：使用 L2 范数（欧几里得范数）
- **统计计算**：使用 NumPy 在后端计算

## 常见问题

**Q: 为什么有些 chunk 没有向量数据？**
A: 可能是文档还在处理中，或者该 chunk 解析失败。

**Q: 向量维度为什么是 1024？**
A: 这取决于你使用的 embedding 模型，不同模型的输出维度不同。

**Q: 如何理解向量范数？**
A: 向量范数表示向量的"长度"，通常在 10-30 之间是正常的。

**Q: 可以修改向量数据吗？**
A: 不可以，向量数据是只读的，由 embedding 模型生成。
